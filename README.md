# Car booking (ตัวอย่างหน้าเว็บจองรถพร้อมคนขับ)

คู่มือนี้อธิบายวิธีรันและใช้งานตัวอย่างเว็บจองรถ (static demo) ที่เก็บข้อมูลการจองไว้ในเบราว์เซอร์ (localStorage)

## ไฟล์สำคัญ

- `index.html` — หน้าเว็บหลัก (ฟอร์มจอง + ส่วนแสดงผล)
- `script.js` — ลอจิกการตรวจสอบ ฟังก์ชันคำนวณราคา และการเก็บ/อ่านข้อมูลจาก `localStorage`
- `server.js` — เซิร์ฟเวอร์ static เล็ก ๆ สำหรับรันด้วย Bun (เลือกใช้หรือไม่ก็ได้)

## วิธีใช้งาน (ผู้ใช้)

วิธีง่ายสุด (ไม่ต้องลงอะไรเพิ่ม):

1. เปิดไฟล์ `index.html` ในเบราว์เซอร์ (ดับเบิลคลิกหรือ Open File)
2. กรอกข้อมูลในฟอร์ม แล้วกด "คำนวณราคา & จอง"
3. ข้อมูลการจองจะถูกแสดงในกล่องผล และถูกบันทึกในเบราว์เซอร์ (localStorage)

รันผ่านเซิร์ฟเวอร์ (แนะนําเมื่อทดสอบในหลายเครื่องหรือใช้ API ต่อไป):

```powershell
# ถ้ามี Bun ติดตั้งแล้ว ให้รันที่โฟลเดอร์โปรเจค
bun server.js
# แล้วเปิด http://localhost:3000/ ในเบราว์เซอร์
```

หมายเหตุ: ถ้าไม่มี Bun คุณสามารถใช้ static server อื่นเช่น `http-server`, `live-server` หรือเปิดผ่าน VS Code Live Server

## รายละเอียดฟิลด์ (สำหรับเข้าใจสคริปต์)

- `customer_name` — ชื่อผู้จอง
- `customer_phone` — เบอร์โทร (สคริปต์ตรวจสอบรูปแบบ: ต้องขึ้นต้นด้วย `0` และเป็นตัวเลข 9–10 หลัก)
- `vehicle_type` — ประเภทรถ (ค่าที่สคริปต์รองรับ: `taxi`, `van`, `executive`)
- `duration_hours` — จำนวนชั่วโมง (ต้องมากกว่า 0)
- `pickup_datetime` — วัน/เวลาที่รับ
- `pickup_address` — สถานที่รับ
- `dropoff_address` — สถานที่ส่ง
- `notes` — หมายเหตุ (ไม่บังคับ)

การคำนวณราคาใน `script.js` ใช้อัตราต่อชั่วโมงในตัว (RATES) และคำนวณจาก `duration_hours * rate` แล้วเก็บเป็น `price` ในอ็อบเจ็กต์การจอง

## เก็บข้อมูลอย่างไร

ข้อมูลการจองจะถูกเก็บเป็น JSON array ใน `localStorage` ภายใต้คีย์ `bookings` ตัวอย่างรูปแบบ:

```json
[
	{
		"id": "BKG-...",
		"customer_name": "...",
		"customer_phone": "0...",
		"price": 1200,
		"created_at": "2025-...",
		...
	}
]
```

## ปัญหาที่พบบ่อย & วิธีแก้

- ฟอร์มไม่ทำงานหรือไม่เห็นผล: ตรวจสอบว่ามี `<script src="/script.js"></script>` อยู่หลังปิดแท็ก `</body>` ใน `index.html` และเปิดหน้าเว็บผ่านเซิร์ฟเวอร์หรือไฟล์
- ข้อความเตือนเกี่ยวกับเบอร์โทร: ให้ใส่เบอร์ที่ขึ้นต้นด้วย `0` และมีความยาว 9–10 หลัก (เช่น `0812345678`)
- localStorage ถูกลบเมื่อเปลี่ยนโฟลเดอร์ / โปรไฟล์เบราว์เซอร์: ให้รันด้วย `bun server.js` หรือใช้เว็บเซิร์ฟเวอร์เสมอเพื่อผลลัพธ์ที่เสถียร

## สำหรับนักพัฒนา / ขยายระบบ

- ต้องการ backend จริง: สร้าง API endpoint (เช่น `POST /api/bookings`) แล้วเปลี่ยน `script.js` ให้ส่งข้อมูลด้วย `fetch()` แทนการเขียน `localStorage`
- เพิ่มหน้า admin: สร้าง UI สำหรับดู/ลบ/ส่งออกการจอง (CSV/JSON)
- เพิ่มระบบ authentication หรือการชำระเงิน: ผสานกับบริการภายนอก (เช่น Stripe หรือระบบธนาคาร)
